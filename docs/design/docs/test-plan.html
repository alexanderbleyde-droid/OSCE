Plexus OSCE Simulator — V3 Test Plan#

Scope: acceptance tests for the V3 design (the station engine). Tests are derived from MASTER_STATION_BLUEPRINT.md and the two reference stations (PMR-002 ITB pump, PLX-PMR-AS-001 Ankylosing Spondylitis).

How to read a test. Each test has an ID, a verification surface, preconditions, numbered steps, and an expected result. Run a test by following the steps exactly and comparing to Expected. Record Pass / Fail / Blocked and a note.

Surface Meaning
[P] Verifiable in the clickable prototype — a screen demonstrates the behaviour.
[E] Engine-acceptance test — must be exercised against the built system; the prototype only illustrates it.

Current prototype status (read first). As of this revision the V3 screens still mirror the V2 baseline; only the scaffold, the Master Station Blueprint, and the launcher have been rebuilt for V3. The Station Report (Screen 02, renamed from "Knowledge Bridge") and the new Knowledge Bridge remediation-engine screen are now built; remaining [P] tests against the exam-chat overhaul and the station-authoring screen are Blocked — pending screen build and are marked accordingly. They are written now so the screens can be checked against them as they land.

A. Difficulty & concealment — the AI Dial#

T-AID-01 — Clinical facts are tier-invariant · [E] Preconditions: ITB station authored; Exam mode. 1. Run the station at Basic, transcribe every clinical fact the simulated patient (SP) states. 2. Re-run at Intermediate, then Advanced, with comparable candidate input. 3. Diff the clinical facts stated across the three runs. Expected: clinical facts, doses, and safety statements are identical across tiers. Only the task prompt, SP affect, and cue explicitness differ. No fact appears in one tier and not another.

T-AID-02 — Concealed task is genuinely hidden · [E] / [P pending] Preconditions: AS station, Advanced tier, Exam mode. 1. Enter the station; read the candidate prompt. 2. Confirm the prompt is the minimal "take a focused history and answer the patient's questions." 3. Confirm the concealed construct (bad news + counselling required) is not shown anywhere in the candidate-facing UI. Expected: construct is hidden in Exam mode; the construct-recognition rubric item exists but is invisible to the candidate; it is scored only on concealed tiers.

T-AID-03 — Per-tier raw totals normalise · [E] 1. Score an AS Senior/Advanced run (max 42) and an AS Basic run (max 40, construct item omitted). 2. Read the percentage on each result. Expected: both report on the same 0–100% wheel; the differing raw maxima do not distort comparability; the construct item is absent from the Basic total.

T-AID-04 — Affect scales with tier · [E] 1. Trigger the same bad-news moment at Basic, Intermediate, Advanced. Expected: Basic SP accepts calmly; Intermediate is mildly anxious; Advanced shows denial/anger/tears and resists — without any change to the underlying facts or rubric.

B. Progressive (controlled) disclosure#

T-DISC-01 — Withheld fact stays locked until elicited · [E] Preconditions: ITB station. 1. Run the station to completion without asking about the patient's concerns or fears. 2. Confirm the "read online that a patient died from pump failure" fear is never volunteered. 3. Re-run; ask an open question about concerns/fears. Expected: the fear is withheld in run 1 and released only after the matching elicitation in run 2.

T-DISC-02 — Disclosure granularity matches question specificity · [E] 1. Ask a broad history question; observe the SP gives one relevant detail, not a data-dump. 2. Ask a targeted question about a single attribute; observe only that attribute is disclosed. Expected: salient facts are not over-disclosed; the SP answers at the granularity asked (the V3 controlled-disclosure rule).

T-DISC-03 — Sensitive concern requires sensitive elicitation · [E] Preconditions: AS, Advanced. 1. Probe the patient's identity/loss concern bluntly. 2. Re-run and explore it sensitively. Expected: the hidden identity/loss concern emerges only when explored well, per the AI Dial.

C. Randomised patient-question pool#

T-POOL-01 — Category floors honoured · [E] 1. Start 20 fresh ITB sessions; for each, log the questions drawn. Expected: every session draws 2–3 questions and always includes ≥1 Safety and ≥1 Lifestyle question; the remaining 0–1 is from any category.

T-POOL-02 — Trigger types · [E] 1. Identify a SPONTANEOUS question and an ON INVITE question. 2. Run a session without ever inviting questions. Expected: the SPONTANEOUS question fires at its defined point; the ON INVITE question does not fire unless the candidate asks "any other questions?".

T-POOL-03 — Check-in scored · [E] 1. Answer a drawn question and omit the comprehension check-in. 2. Re-run and include "does that answer your question?". Expected: the check-in item scores 0 in run 1, full in run 2.

D. Jargon detection & communication layer#

T-JARG-01 — First flag, no penalty · [E] 1. Use a flagged term (e.g. "intrathecal") with no same-sentence lay translation. Expected: SP reacts in character with confusion; no deduction yet; candidate gets a chance to self-correct.

T-JARG-02 — Escalation curve · [E] 1. Use 2 unexplained flagged terms → check SP mild frustration + minor comm deduction. 2. 3 terms → visible disengagement + moderate deduction + comm score capped. 3. 4+ terms → SP stops engaging; teach-back is set to fail; comm cannot recover. Expected: behaviour and scoring match the 1/2/3/4+ table for this station.

T-JARG-03 — Self-correction resets; same-turn new flag does not · [E] 1. Use a flag, then self-correct after the SP prompt → counter resets, no deduction. 2. Self-correct one term but introduce a new flagged term in the same turn. Expected: run 1 resets to 0; run 2 does not reset.

T-JARG-04 — Morphological + case-insensitive matching · [E] 1. Use "titrating" and "Titrated" (flag is "titrate"). 2. Use "Intrathecal" capitalised. Expected: all variants flag; matching is case-insensitive and morphology-aware.

T-JARG-05 — Acceptable use does not flag · [E] 1. Say "…delivered into the fluid around your spinal cord — what we call intrathecal." Expected: no flag (term + same-sentence plain-language translation is acceptable).

T-JARG-06 — Prototype shows the counter · [P pending] 1. Open exam-chat (01); reproduce a jargon flag. Expected: the flag reaction and an escalating counter are visible in the prototype. (Blocked — pending screen 01 build.)

E. Closing protocol & teach-back#

T-CLOSE-01 — No teach-back is a silent critical fail · [E] 1. Close the station without asking the patient to summarise. Expected: the SP does not prompt for teach-back; the station records a critical fail (teach-back is a critical safety item).

T-CLOSE-02 — Missing brochure/contact prompts the SP · [E] 1. Attempt to close without offering written material / contact number. Expected: the SP asks spontaneously ("will you give me anything written?" / "how do I reach you?").

T-CLOSE-03 — Teach-back is evaluated, not assumed · [E] 1. Ask for a teach-back; have the SP return an incorrect summary. Expected: the engine evaluates the SP's generated summary and only credits the item if the candidate actually elicited correct understanding.

F. Exam mode vs Tutor mode#

T-MODE-01 — Exam mode reveals nothing · [E] 1. Stall mid-encounter in Exam mode. Expected: no coaching, no hints, no construct reveal; assessment is silent.

T-MODE-02 — Tutor nudges only at natural pauses · [E] 1. In Tutor mode, pause after finishing a phase → expect a Socratic nudge. 2. Pause mid-sentence → expect no interruption. Expected: nudges fire only at natural pauses, never mid-sentence, and never give the answer outright.

T-MODE-03 — Tutor post-feedback is directive · [E] 1. Finish a Tutor run that missed the construct. Expected: feedback reveals the concealed construct, maps performance to specific rubric items, names any bridge fired, and gives one or two concrete improvements (not an exhaustive list).

G. Knowledge Bridge — remediation engine#

T-BRDG-01 — Domain <50% fires a bridge · [E] 1. Score a run with one domain subtotal below 50%, others passing. Expected: a bridge fires for that domain and routes to its named drill.

T-BRDG-02 — Critical-fail item fires regardless of aggregate · [E] 1. Score 0 on a critical-fail item while keeping the overall percentage above threshold. Expected: the station still fails; the critical bridge fires; the failure is surfaced explicitly, not averaged away.

T-BRDG-03 — Construct miss fires on concealed tiers · [E] 1. On Advanced, score the construct-recognition item 0. Expected: the construct bridge fires (construct/prioritisation drill).

T-BRDG-04 — Lock, route, return on confidence path · [E] 1. Trigger a critical-safety bridge (e.g. AS spinal fragility). Expected: the main station locks; the candidate is routed to the correct drill (spinal-fragility safety drill); on return they get a confidence path that does not immediately re-test the failed skill in isolation; critical-safety bridges take priority.

T-BRDG-05 — Station Report surfaces the outcome · [P] 1. Open the Station Report (Screen 02) with a failing sample. Expected: the score wheel, the pass/threshold outcome, narrative feedback and any critical-safety flags are shown, and the page hands off to the Knowledge Bridge. The Station Report is a summary — it contains no remediation content of its own.

T-BRDG-06 — Knowledge Bridge generates targeted reinforcement · [P] 1. From the Station Report, open the Knowledge Bridge (the remediation-engine screen). Expected: a gap analysis names the weak domains and any critical-safety gap; a generated reinforcement plan presents content tagged by type — mini-case, MCQ set, clinical pearl, framework review — each targeted to a specific weak domain; a confidence-path return follows. The Knowledge Bridge is a distinct learning surface, not the results page.

H. Scoring integrity (per-station domains)#

T-SCORE-01 — Wheel renders the active station's own domains · [P pending] / [E] 1. Open the result for ITB; note the five domain labels (Data Gathering, Clinical Reasoning & Counseling, Communication, Safety, Professionalism & Organisation). 2. Open the result for AS; note its labels (Setup/Construct Recognition, Breaking Bad News, Disease & Safety Knowledge, Counselling & Alternatives, Communication/Integration). Expected: the wheel shows each station's own domain set; the label sets differ between stations. The score wheel renders on the Station Report; per-station variation is illustrated across the ITB and AS station pages.

T-SCORE-02 — Critical-fail override · [E] Expected: a 0 on any critical-fail item fails the station even when the aggregate percentage is at or above threshold; the override is shown explicitly on the result.

T-SCORE-03 — Threshold default + configurability · [E] 1. Confirm default pass threshold is 65%. 2. Score AS at exactly 27/42 (≈64.3%). 3. Reconfigure the station threshold to 60%. Expected: 65% default; the AS boundary behaves per the configured value; threshold is settable per station/programme.

T-SCORE-04 — Level-adaptive weighting · [E] 1. Score the same raw performance as student, then resident. Expected: the active station's domains re-weight per level; the percentage shifts accordingly; tier (concealment) and level (weighting) remain independent but coupled by default.

I. Cross-cutting#

T-X-01 — Clinical-content lock (hallucination guard) · [E] 1. Ask the SP a question whose answer is not in the station's locked content. Expected: the SP deflects or stays within locked facts; it never invents a clinical detail, dose, or safety claim. AI may rephrase tone only. (Also a compliance requirement — see STATION_COMPLIANCE.md.)

T-X-02 — Theme toggle · [P] 1. On each screen, toggle dark/light (top-right). Expected: both themes render correctly; brand colours and the gold pass-mark accent hold; no contrast breakage.

T-X-03 — Standalone render + relative links · [P] 1. Open each screen file directly; from the launcher, click every V3 link. Expected: each screen renders standalone; all internal links resolve (no ../, no broken targets).

T-X-04 — Model routing · [E] Expected: simulation turns use Claude Haiku 4.5; case generation, scoring, and Knowledge Bridge use Claude Sonnet 4.6 (per IMPLEMENTATION_PLAN.md).

J. Per-station end-to-end smoke runs#

T-E2E-ITB-PASS · [E] — Run an ideal ITB encounter: rapport → context → prior-treatment review → ITB mechanism in plain language → benefits tied to holding his son → all four risk categories → contraindication screen → trial-dose → full withdrawal warning with ER instruction + pump-ID card + the oral-baclofen-can't-rescue point → teach-back → brochure/contact/follow-up/thanks. Expected: pass with no critical fail; no jargon cap.

T-E2E-ITB-FAIL · [E] — Run the same but omit the withdrawal warning. Expected: critical fail regardless of aggregate; Safety bridge → ER-withdrawal mini-case.

T-E2E-AS-PASS · [E] — Advanced tier: recognise the construct unprompted → warning shot → deliver plainly → pause for emotion before facts → counsel on exercise/posture/joint-protection → offer safe alternatives → shared plan + MDT → neuro safety-net. Expected: construct item scored; pass; no critical fail.

T-E2E-AS-FAIL · [E] — Advanced tier but miss spinal fragility / do not ban contact sport. Expected: item 8 critical fail; spinal-fragility safety bridge (highest priority); return path re-tests the safety item at the next tier up.
